主要 frontend/apps/dashboard/src/pages/login/login-page.tsx:31-47
页面初始化时会同时触发 `ssoConfig()` 和基于 URL `token` 的 `ssoExchange()`。当登录页是从已登录状态进入，且本地已有旧 token 时，`ssoConfig()` 请求会先带着旧 `Authorization` 发出；若后端因此返回 401，现有全局 401 处理通常会清空登录态，而随后 `ssoExchange()` 才写入新的 SSO 登录态。这样会造成一次不必要的登出/重登竞态，表现为 SSO 回跳后偶发闪退、状态被清空或跳转异常。
💡 修复建议：在处理 `token`/`ssoError` 的 effect 中优先完成兑换，再决定是否拉取 `ssoConfig`；或者当 URL 中存在 `token`/`ssoError` 时跳过 `ssoConfig()` 请求。示例：`useEffect(() => { if (searchParams.get('token') || searchParams.get('ssoError')) return; authApi.ssoConfig()... }, [searchParams]);`。
