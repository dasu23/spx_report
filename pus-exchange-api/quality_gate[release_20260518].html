主要 pus-domain/src/main/java/io/speedx/pus/domain/core/ThreadPoolOrchestrator.java:146-149
`Runnable` 任务执行时对 `OutOfMemoryError` 和普通 `Exception` 只记录日志不再抛出。由于外层使用的是 `executor.submit(coreTask)`，这些异常被吞掉后，返回的 `Future` 会表现为正常完成，调用方无法通过 `Future.get()` 感知任务失败，容易造成任务失败被误判为成功。
