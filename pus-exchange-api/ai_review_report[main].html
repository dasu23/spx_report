major5配置变更在通用 `bootstrap.yml` 中将 `spring.profiles.active` 固定为 `dev`，会让应用在未显式覆写启动参数时默认加载开发环境配置。对发布包、CI 或临时环境来说，这会把环境选择从部署侧挪到代码侧，容易误连开发配置或跳过预期的生产配置。删除该默认值，改由部署环境通过启动参数或环境变量注入，例如移除 `active: dev`，在启动时使用 `--spring.profiles.active=prod` 或通过 `SPRING_PROFILES_ACTIVE` 设置。在未传入任何 profile 参数的情况下启动应用，确认不会默认落到 `dev`；分别在 dev/prod 部署脚本中显式设置 profile，验证加载的配置文件符合预期。
