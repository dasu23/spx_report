    这里的分支条件与方法注释和实现目标相反。当前代码对 `http://`、`https://` 或 `speedx.io` 域名的 URL 直接原样返回，导致这类完整 Blob 地址不会重新生成 SAS token，也不会替换为配置的 `blobDomain`，刷新逻辑实际失效。

    测试场景

    修复建议

    不要用字符串首个 `/` 判断 host，改为基于 URI/URL 解析主机名，例如 `URI uri = URI.create(url); String host = uri.getHost(); boolean isSpeedxDomain = StringUtils.endsWith(host, "speedx.io");`。同时兼容无 scheme 的相对路径场景。

    测试场景

    补充用例：`https://cdn.speedx.io/a/b.png?x=1` 应识别为 `speedx.io` 域名；`/container/a.png` 和 `container/a.png` 不应走域名判断分支。
