    MAJORL70-152API契约
    `Runnable` 任务包装器捕获 `OutOfMemoryError` 和普通 `Exception` 后仅记录日志、不再向外抛出，导致 `executor.submit(coreTask)` 返回的 `Future` 会表现为“执行成功”。调用方无法通过 `Future.get()` 感知任务失败，与同文件 `Callable` 重载会向上传播异常的行为不一致。这会让上层的失败告警、重试熔断或补偿逻辑失效，故障任务可被持续静默放大，带来可用性风险。

    修复建议
