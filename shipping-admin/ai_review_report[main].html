    MAJORL172-179前端安全
    预览文件时直接对服务端返回的 URL 执行 `window.open(..., '_blank')`，未设置 `noopener,noreferrer`。如果返回地址可跳转到恶意页面，新页面可通过 `window.opener` 控制原后台页面，形成 tabnabbing / opener 劫持风险。
    修复建议与测试场景
    修复建议

    改为 `window.open(url, '_blank', 'noopener,noreferrer')`，或先打开空白页并显式将 `newWindow.opener = null`；同时限制预览 URL 仅允许受信任域名。

    测试场景

    构造一个返回恶意 HTML 页面的预览 URL，在新标签页中尝试访问 `window.opener.location`；修复后应无法获取或操控 opener。

    MAJORL64-64边界条件
    将 `value` 的默认值从空数组改为 `undefined` 后，后续上传成功分支仍直接执行 `[...]value]`。在组件以多文件模式且未传初始值时，首次上传会因展开 `undefined` 抛出运行时异常。
    修复建议与测试场景
    修复建议

    为 `value` 提供稳定的数组兜底，或在拼接前归一化，例如 `const currentValue = Array.isArray(value) ? value : []; const newValue = multiple ? [...currentValue, entryValue] : entryValue;`。

    测试场景

    以 `multiple=true` 且不传 `value` 挂载组件，上传首个文件，确认不会报 `value is not iterable`，并能正确回填文件列表。
