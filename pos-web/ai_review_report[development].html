src/pages/user/sso/index.tsx

SeverityLineCategoryDescriptionSuggestionTest Scenario
major10-19安全风险新的 SSO 回调页直接信任 URL 中的 `authToken` 并写入 `localStorage`，没有校验任何一次性 `state`/nonce，也没有验证这次回调是否由当前登录流程发起。攻击者只要诱导用户访问 `/user/sso?authToken=<attacker_token>`，就可以把受害者浏览器会话固定到任意账号，形成典型的登录 CSRF / session fixation 风险。发起 Microsoft 登录前生成随机 `state` 并写入 `sessionStorage`，回调时同时校验 `state` 与 `authToken`，校验失败时拒绝写入 token。例如：`const state = crypto.randomUUID(); sessionStorage.setItem('sso_state', state);`；回调页中读取 `urlParams.get('state')` 与 `sessionStorage.getItem('sso_state')` 比较，一致后再 `localStorage.setItem(SPX_AUTH_TOKEN, authToken)`，随后立即清理 `state`。手工访问 `/user/sso?authToken=fake-token&state=fake`，应被拦截且不会覆盖现有登录态；再走一次正常 SSO 流程，确认合法回调仍可登录成功。
