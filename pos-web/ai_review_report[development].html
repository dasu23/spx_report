SeverityLineCategoryDescriptionSuggestionTest Scenario
major15-18业务逻辑SSO 回调成功后只执行 `localStorage.setItem + window.location.href='/'`，没有处理 popup 场景（`window.opener.postMessage` + `window.close`）。这与同次变更中的测试预期（`src/pages/user/sso/__test__/index.test.tsx`）不一致，实际在弹窗登录链路下会导致登录页无法收到成功信号。在回调页增加双模式处理：若存在 `window.opener` 且同源，则 `postMessage({ type: 'MICROSOFT_LOGIN_SUCCESS', authToken })` 并 `window.close()`；否则走当前的本页落地逻辑（写 token 后跳转首页）。错误分支同理发送 `MICROSOFT_LOGIN_ERROR`。1) 弹窗方式打开 Microsoft 登录，回调到 `/user/sso?authToken=...` 后验证 opener 收到消息且弹窗关闭；2) 直接访问 `/user/sso?authToken=...` 时仍能写入 token 并跳转 `/`；3) `/user/sso?error=...` 在弹窗/直开两种模式都能正确处理。
