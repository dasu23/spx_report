    src/pages/user/sso/index.tsx1 Critical1 个问题
    CRITICALL18安全风险
    将 `authToken` 直接写入 `localStorage`。认证令牌属于高敏感凭证，落地到 Web Storage 后可被任意同源脚本读取；一旦站点存在 XSS、第三方脚本污染或浏览器扩展窃取，令牌会被直接导出并导致会话接管。该风险也命中审查清单中的“敏感数据暴露：Token 存在 localStorage/sessionStorage/cookie”与自定义规则 `[SEC-002]`。
    修复建议与测试场景
    修复建议

    不要在前端用 `localStorage`/`sessionStorage` 持久化认证 Token。改为由后端在 SSO 回调成功后下发 `HttpOnly + Secure + SameSite` 的会话 Cookie；如果架构上必须前端暂存，也应仅保存在内存态，并配合短时效一次性码换取服务端会话。同步检查请求头注入逻辑，避免继续从 Web Storage 读取令牌。

    测试场景

    新增安全回归测试：1. 完成 SSO 回调后断言 `localStorage`/`sessionStorage` 中不存在认证 Token；2. 断言后续鉴权请求依赖 `HttpOnly` Cookie 或内存态而非 Web Storage；3. 在浏览器控制台执行 `localStorage.getItem('SPX_AUTH_TOKEN')` 应返回 `null`。
