major10-19安全风险SSO 回调页直接从 URL 读取 `authToken` 并写入 `localStorage`，没有校验任何 `state`/nonce/来源信息。攻击者只要构造 `/user/sso?authToken=<their-token>` 链接并诱导用户访问，就能把受害者浏览器会话固定到攻击者账号，属于典型的登录 CSRF / session fixation 风险。不要信任 URL 里的成品 token。发起登录前生成并保存随机 `state`，回调时先校验 `state`；更稳妥的做法是让回调页只接收一次性 `code`，再由后端换取 token，并由后端/HttpOnly Cookie 建立会话。至少在前端落 token 前增加对 `state` 的严格校验，不通过则拒绝登录。1. 正常发起 Microsoft 登录并带回合法 `state`，应成功登录。2. 直接访问 `/user/sso?authToken=fake-token` 或携带错误 `state`，页面应报错且不能写入本地 token。
