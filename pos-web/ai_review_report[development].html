    config/routes.ts1 Major1 个问题
    MAJORL108-342权限与访问控制
    **原因：** 新增 `/tms-pda` 路由组及其 5 个子页面时，所有新增页面均未配置 `access` 权限控制，且文档中明确写了“5 个功能暂不加权限码（Home 宫格默认全员可见）”。这意味着任何已登录且能触达前端路由的用户都可直接访问这些 PDA 操作页面，而不是仅限被授权的 TMS/PDA 操作人员。 ```ts { path: '/tms-pda', ... routes: [ { path: 'loading-scan', component: './tms-pda/loading-scan' }, { path: 'unloading-scan', component: './tms-pda/unloading-scan' }, { path: 'seal-scan', component: './tms-pda/seal-scan' }, { path: 'unseal-scan', component: './tms-pda/unseal-scan' }, { path: 'abnormal-report', component: './tms-pda/abnormal-report' }, ], } ``` **影响：** 未获授权的普通用户可直接打开装车、卸车、封签、解签、异常上报等高敏感作业页面，扩大业务操作面；若对应页面调用后端接口时仅依赖前端入口收敛，可能进一步导致越权提交扫描/异常数据。
    修复建议与测试场景
    修复建议

    为 `/tms-pda` 父路由及每个子路由补充明确的 `access` 权限码，并同步在首页快捷入口、菜单搜索等所有入口复用同一权限判断。例如： ```ts { path: '/tms-pda', access: 'tms:pda', routes: [ { path: 'loading-scan', access: 'tms:pda:loadingScan', ... }, { path: 'unloading-scan', access: 'tms:pda:unloadingScan', ... }, ... ], } ``` 同时确认后端对应接口也做服务端鉴权，避免仅靠前端路由限制。

    测试场景

    使用无 PDA 权限的普通账号，直接访问 `/tms-pda/loading-scan`、`/tms-pda/abnormal-report` 应被拒绝或重定向；有权限账号应可正常访问。
