major39配置变更`content` 只扫描了 `pages/components/layouts` 下的 `*.tsx`，会遗漏 `src/app.tsx`、`src/global.tsx`、`hooks`、`utils` 或 `*.ts` 文件中的 Tailwind 类名。生产构建时这些未被扫描到的类会被裁掉，造成样式在 build 后缺失。把扫描范围扩到整个 `src`，至少覆盖 `ts/tsx/js/jsx`，例如：`content: ['./src/**/*.{js,jsx,ts,tsx}']`；如果确实要限制范围，也需要显式补上 `src/app.tsx` 等入口文件。执行生产构建后检查 `src/app.tsx` 或其他非 `pages/components/layouts` 文件里使用的 Tailwind 类是否仍然出现在最终 CSS 中，并确认对应页面样式未丢失。
