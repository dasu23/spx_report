major39配置变更`content` 只扫描 `src/pages`、`src/components` 和 `src/layouts`，会把 `src/app.tsx`、`src/global.tsx` 及其他目录里的 Tailwind 类名全部排除在生成结果之外。此次变更文件列表里已经包含 `src/app.tsx`，因此这些文件中的工具类在生产构建中会直接失效，表现为页面样式缺失。把扫描范围至少扩展到整个 `src`，例如：`content: ['./src/**/*.{ts,tsx}']`；如果确实要做白名单，也要显式补上 `src/app.tsx`、`src/global.tsx` 等实际使用 Tailwind 类的入口文件。执行一次生产构建后，验证 `src/app.tsx` 和非 `pages/components/layouts` 目录组件中的 Tailwind 类是否仍然出现在最终 CSS 中，并检查对应页面样式没有丢失。
minor38向后兼容全局启用 `important: true` 会把所有 Tailwind 工具类提升为 `!important`，这很容易覆盖 Ant Design/Pro Components 现有样式优先级，导致按钮、表单、布局等组件在交互态或主题态下出现不可预期的样式回归。除非已经完成全量回归验证，否则不要在现有 Ant Design 项目里全局开启 `important`。优先改为局部容器方案，例如 `important: '#root'`，或仅在确有冲突的组件上提升优先级。回归检查使用 Tailwind 与 Ant Design 混排的页面，重点验证按钮 hover/disabled、表单校验态、Drawer/Modal 内部布局样式是否被 Tailwind 工具类意外覆盖。
