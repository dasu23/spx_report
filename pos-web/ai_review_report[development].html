    INFOL155-183权限与访问控制
    **原因：** 新增的 `tmsLoadingUnloading` 快捷组下 5 个入口均未配置 `rightCode`，代码注释还明确写了“暂不设 rightCode（全员可见）”。这会让首页快捷入口绕过既有的权限过滤逻辑（`filterShortcuts` 只能基于 `rightCode` 做裁剪），导致任何能访问首页的用户都能看到并尝试进入装卸车相关页面。 ```tsx { key: 'tmsLoadingScan', path: '/tms-pda/loading-scan', ... // 权限码待后续开会确定，暂不设 rightCode（全员可见） }, ... { key: 'tmsAbnormalReport', path: '/tms-pda/abnormal-report', ... } ``` **影响：** 如果对应路由页或后端接口未做同等严格鉴权，低权限用户可直接发现并访问装车、封车、解车、卸车、异常上报等操作入口，扩大攻击面；即使后端最终拦截，也会造成功能枚举与未授权访问尝试窗口，属于新增权限暴露。
    修复建议与测试场景
    修复建议

    为每个 TMS PDA 快捷入口补充明确的 `rightCode`，并确保与路由/页面访问控制一致；在权限码未确定前，不应默认全员展示，建议先隐藏该组或加临时受控开关。例如： ```tsx { key: 'tmsLoadingScan', path: '/tms-pda/loading-scan', titleKey: 'menu.tmsPda.loadingScan', icon: <CarOutlined />, color: '#f5222d', rightCode: 'tmsPda:loading:scan', } ``` 同时补充页面级守卫，避免仅靠首页快捷入口控制可见性。

    测试场景

    使用无 TMS PDA 权限的账号登录：1）首页不应出现 `tmsLoadingUnloading` 组；2）直接访问 `/tms-pda/loading-scan`、`/tms-pda/seal-scan`、`/tms-pda/unseal-scan`、`/tms-pda/unloading-scan`、`/tms-pda/abnormal-report` 时应被前端路由守卫或后端鉴权拒绝。

    src/services/tms-pda/request.ts1 Critical1 个问题

    CRITICALL31-33安全风险
    PDA 专用请求实例把认证凭证直接从 `localStorage` 读取并附加到所有请求头：`Authorization: localStorage.getItem(SPX_AUTH_TOKEN) || ''`。一旦站内任意位置出现 XSS，攻击脚本即可直接读取该 Token 并发往外部，导致会话劫持；这属于新增的敏感凭证暴露面。
    修复建议与测试场景
    修复建议

    避免将长期有效的认证 Token 存放在 `localStorage`。优先改为 HttpOnly + Secure Cookie，或至少改为仅内存态保存并通过受控刷新机制续期。若短期内无法调整后端鉴权方式，需补充严格的 CSP、统一 XSS 防护，并封装单点读取逻辑避免在多个新模块继续扩散。

    测试场景

    在浏览器控制台模拟 `localStorage.getItem(SPX_AUTH_TOKEN)` 可直接取到有效凭证；再构造一个最小化 XSS PoC（如页面某处可注入脚本时）验证是否能读取并外传该 Token。
