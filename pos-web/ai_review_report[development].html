major38配置变更将 `important` 全局设为 `true` 会把所有 Tailwind 工具类提升为 `!important`。当前项目以 Ant Design Pro/Ant Design 为主，这会直接改变与组件库样式的优先级关系，常见结果是按钮、表单、表格间距/颜色等样式被意外覆盖，属于高风险的全局 UI 回归。避免使用全局 `important: true`。如果只是想提升局部容器内 Tailwind 样式优先级，改为作用域选择器，例如 `important: '#root'` 或某个业务容器；如果确实需要覆盖，优先在具体组件上定向处理，而不是全局提升。回归检查 Ant Design 组件页：重点验证 `Button`、`Form`、`Table`、`Modal`、`ProTable` 在有 Tailwind class 的页面中是否出现字号、颜色、padding、margin、display 被覆盖的情况。
major39配置变更`content` 仅扫描 `src/pages/**/*.tsx`、`src/components/**/*.tsx`、`src/layouts/**/*.tsx`，会漏掉当前仓库中的其它 Tailwind 使用位置，例如 `src/app.tsx`、hooks/工具函数中拼接出来的 class，甚至后续新增到 `src/features`、`src/modules` 等目录的组件。生产构建时这些未被扫描到的类会被清理，导致页面样式缺失。将扫描范围至少扩展到 `src/**/*.{ts,tsx}`，例如：`content: ['./src/**/*.{ts,tsx}']`。如果担心性能，再按实际目录做白名单，但需要覆盖所有可能产出 JSX/className 的位置。执行一次生产构建后的页面回归，重点检查 `src/app.tsx` 引入的布局区、动态 className 组件以及非 `pages/components/layouts` 目录下的页面，确认 Tailwind 样式没有在构建产物中丢失。
