    src/pages/user/sso/index.tsx1 Critical1 个问题
    CRITICALL10-17安全风险
    SSO 回调直接从 URL query 中读取 `authToken`。将访问令牌放在查询参数里会泄露到浏览器历史、反向代理/服务器日志、监控链路以及后续跳转的 Referer，属于敏感凭证暴露。
    修复建议与测试场景
    修复建议

    不要在 URL query 中传递登录令牌。改为标准 OAuth/OIDC code flow：回调仅携带一次性 `code`，前端再通过后端安全换取会话；如果必须前端接收结果，至少使用短时一次性票据并通过 `postMessage`/`window.opener` 传递，避免令牌出现在地址栏。

    测试场景

    构造 `/user/sso?authToken=test-token` 回调，验证浏览器地址栏、代理访问日志、埋点日志和后续外链 Referer 中都不会出现真实登录令牌。
