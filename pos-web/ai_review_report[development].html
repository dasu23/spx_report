major41配置变更全局开启 `important: true` 会把所有 Tailwind utility 提升为 `!important`。当前项目大量依赖 Ant Design / Pro Components，这个变更容易直接覆盖组件库的状态样式、布局样式和运行时注入样式，造成按钮、表单、表格等 UI 表现异常。避免全局开启 `important: true`。如果只是想提升局部优先级，改用命名空间方式，例如 `important: '#root'`，或仅在确有冲突的少量样式上手动使用 `!` 前缀类。回归检查 Ant Design 组件的 hover/disabled/error/status 样式，特别是 ProTable、Form、Modal、Button 等场景，确认 Tailwind utility 没有异常覆盖组件库样式。
major42配置变更`content` 只保留了 `src/pages`、`src/components`、`src/layouts` 三个目录，Tailwind 将不再扫描 `src/app.tsx`、`src/global.tsx` 以及其它 `src/**` 下的 TS/TSX 文件。若这些位置存在 Tailwind 类名，生产构建会直接被 purge 掉，导致页面样式缺失。将扫描范围恢复为覆盖整个 `src`，或至少补齐当前实际使用 Tailwind 的目录/文件，例如：`content: ['./src/**/*.{ts,tsx}']`。如果必须收窄范围，先确认 `src/app.tsx`、`src/global.tsx`、hooks/业务模块中不存在动态或静态 Tailwind 类名。执行生产构建后，重点检查 `src/app.tsx` 和非 `pages/components/layouts` 目录内使用 Tailwind 的页面，确认类名未丢失、样式正常渲染。
