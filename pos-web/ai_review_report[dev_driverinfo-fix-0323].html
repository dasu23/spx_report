SeverityLineCategoryDescriptionSuggestionTest Scenario
major39前端安全启用 `important: true` 会把所有 Tailwind 工具类提升为 `!important`。在当前项目大量使用 Ant Design 的前提下，这会全局改变样式优先级，极易覆盖组件库的禁用态、校验态、交互态样式，造成整站级 UI 回归。不要全局开启 `important: true`。如果只是避免局部样式冲突，改为限定作用域，例如 `important: '#root'`，或仅在冲突组件处增加更精确的 class/样式覆盖。回归验证 Ant Design 表单校验、按钮 disabled、Modal、Table、Select 下拉高亮等交互态样式，确认没有被 Tailwind 工具类强制覆盖。
minor40配置变更`content` 只扫描 `pages/components/layouts` 下的 `tsx` 文件，会遗漏 `src/app.tsx` 以及其它目录/后缀中的 Tailwind class，生产构建时这些类会被 purge 掉，表现为部分页面样式在本地正常但上线后丢失。扩大扫描范围，至少覆盖整个 `src` 下常见前端文件，例如 `content: ['./src/**/*.{ts,tsx}']`；如果明确只想限制范围，也要把 `src/app.tsx` 等入口文件补进去。执行生产构建后检查使用 Tailwind 的入口布局、登录页、全局容器等是否仍保留样式，重点验证 `src/app.tsx` 中引用的 class 没有丢失。
