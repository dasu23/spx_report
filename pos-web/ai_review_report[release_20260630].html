    src/hooks/usePrintWithFallback.tsx1 Major1 个问题
    MAJORL31-34安全风险
    打印机连接信息从 `localStorage` 的 `print_ip` 直接读取并 `JSON.parse`，且后续在 `onFinish` 中持续写回本地存储。该实现把设备网络信息长期暴露在可被任意同源脚本读取的位置；一旦站点存在 XSS/第三方脚本被污染，攻击者可枚举内网打印机地址并用于横向探测或误打单。
    修复建议与测试场景
    修复建议

    避免将打印机 IP 等设备敏感信息持久化到 `localStorage`。优先改为内存态或 `sessionStorage`，并对读取内容做 `try/catch` 与结构校验；如必须持久化，至少最小化字段并增加过期控制。示例：`const raw = sessionStorage.getItem('print_ip'); let parsed = null; try { parsed = raw ? JSON.parse(raw) : null; } catch { parsed = null; }`

    测试场景

    在浏览器控制台预置恶意/异常 `print_ip` 值，验证页面不会泄露或长期保留打印机地址；同时验证刷新会话后敏感配置自动失效。
