    rps-service/src/main/java/io/speedx/rps/service/bo/RpsAddressBo.java1 Minor1 个问题
    MINORL20-24业务逻辑
    `normalizeAddress12` 在判空前没有先做 `trim`。当 `address1` 有前后空格、`address2` 为空或仅包含空白时，会直接 `return`，导致 `address1` 的空格无法被清理；这与方法注释里“Trim address1/address2”的语义不一致，也会让仅填写单行地址的场景继续保留脏数据。
    修复建议与测试场景
    修复建议

    先对两个字段做 `trimToNull`，再执行后续判空和交换逻辑，例如：`String a1 = StringUtils.trimToNull(addr.getAddress1()); String a2 = StringUtils.trimToNull(addr.getAddress2()); addr.setAddress1(a1); addr.setAddress2(a2); if (a1 == null || a2 == null) return;`。这样即使只有一行地址，也能完成基础清洗。

    测试场景

    补充用例：`address1 = " 123 Main St ", address2 = " "`，调用后应得到 `address1 = "123 Main St"`、`address2 = null`。
