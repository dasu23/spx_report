    MAJORL56边界条件
    新增的 `host.startsWith(...)` 判定在 `HOST` 为 `null` 时会直接触发 `NullPointerException`。旧逻辑对 `HOST` 为 `null` 虽然同样是不合法配置，但只会拼出 `redis://null:PORT`，不会在 Spring 初始化阶段因空指针直接崩溃；这次修改引入了更早、更隐蔽的启动失败风险。
    修复建议与测试场景
    修复建议

    在调用 `startsWith` 前先做空值校验，并对非法配置给出明确异常，例如：`if (StrUtil.isBlank(HOST)) { throw new IllegalArgumentException("redis host must not be blank"); } String host = HOST; if (!host.startsWith("redis://") && !host.startsWith("rediss://")) { host = "redis://" + host; }`。这样既避免空指针，也能让配置错误更容易定位。

    测试场景

    补充配置边界测试：将 `HOST` 设为 `null` 或空串时，验证配置初始化不会抛出 `NullPointerException`，而是返回明确的参数校验异常。
