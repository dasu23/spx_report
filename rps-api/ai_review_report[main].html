    MINORL291-296性能隐患
    `readLocalJson()` 使用 `Files.readAllBytes` 将整个文件一次性读入内存，随后又执行一次 `JSONUtil.parse(json)` 全量解析，相当于对同一份内容做两次大对象加载。文件稍大时会明显放大堆内存占用，容易在压测或误放入大文件时触发内存抖动。
    修复建议与测试场景
    修复建议

    至少增加文件大小上限校验，并避免“全量读取 + 再全量解析”的双重开销。可改为先用 `Files.size(filePath)` 做上限控制，再使用流式读取/解析；如果只是校验 JSON，可直接基于输入流解析而不是先构造完整字符串。
