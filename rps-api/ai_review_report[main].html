    MAJORL79-90安全风险
    新增的落盘逻辑将 `trackingNumber` 直接拼入文件名（`fileName = trackingNumber + "_" + uuid`），再经 `WriteUtils.get*FilePath(...)` 生成目标路径后直接 `Files.write(...)`。若 `trackingNumber` 可被外部输入影响且未在 `WriteUtils` 内做严格白名单清洗，可能引入路径穿越/非法文件名写入风险，导致越界写文件或覆盖非预期文件。
    修复建议与测试场景
    修复建议
