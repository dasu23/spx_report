严重 backend/src/main/java/com/aidpm/common/security/JwtAuthenticationFilter.java:73-76
权限是直接从 JWT 中的 `role` claim 生成的，而这里虽然已经查库拿到了最新 `User`，却没有用数据库中的当前角色做校验或覆盖。只要用户角色被后台降权但 `tokenVersion` 未同步递增，旧 token 仍会继续携带并生效原有高权限角色，形成越权访问。
💡 修复建议：认证后应以数据库中的当前角色为准构造权限，或在角色变更时强制递增 `tokenVersion` 使历史 token 全部失效。示例：`new SimpleGrantedAuthority("ROLE_" + user.getRole())`，并同时校验 token 中角色与数据库角色一致，不一致时返回 `TOKEN_INVALID`。

主要 backend/src/main/java/com/aidpm/auth/service/AuthService.java:29-31

主要 backend/src/main/java/com/aidpm/common/exception/GlobalExceptionHandler.java:21-28

💡 修复建议：将校验异常改为返回 `HttpStatus.BAD_REQUEST`。示例：`return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(Result.error(ErrorCode.INVALID_PARAM, message));`。
