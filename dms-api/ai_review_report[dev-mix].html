    修复建议

    为实体补充明确的表名并与 `@SQLDelete` 保持一致，例如：`@Table(name = "parcel")`，同时将 SQL 改为 `@SQLDelete(sql = "update parcel set deleted = id where id = ?")`。

    新增一个 Parcel 记录后调用 JPA delete，验证生成的软删除 SQL 能成功执行，且记录随后被 `@Where(clause = "deleted = 0")` 过滤掉。

    `status` 和 `trackingEnd` 是枚举字段，但未显式声明 `@Enumerated(EnumType.STRING)`。JPA 默认按 ordinal 持久化，后续一旦调整枚举顺序或插入新值，会造成历史数据语义错乱。
    修复建议与测试场景
    修复建议
