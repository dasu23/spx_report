主要 drivermgmt-infrastructure/src/main/java/io/speedx/drivermgmt/integration/AzureBlobUrlSigner.java:140
`crossContainerClients` 是进程级无界缓存，key 直接来自新增解析逻辑中的 `/{container}/{blob}` 输入。只要调用方持续传入不同的 container 名，就会在 `computeIfAbsent` 中永久保留新的 `BlobContainerClient`，造成堆内存持续增长，最终可被利用为应用层 DoS。该风险由本次改动新增，因为旧实现只使用固定注入的 `containerClient`，不会按用户输入扩容缓存。
