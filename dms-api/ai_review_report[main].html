    `rejectRoute` 在事务外先读取并校验 `Route`/`RouteDispatch` 状态，随后才在 `transactionTemplate.execute(...)` 中更新。若同一条路线同时发生“接受路线”或重复“拒绝路线”请求，两个线程都可能基于相同的旧状态通过校验，最终把已被其他线程接受的指派覆盖成 `REJECTED/NEW`，造成状态竞争和错误轨迹。
