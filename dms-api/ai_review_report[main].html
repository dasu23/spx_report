    MAJORL5-8数据一致性
    脚本删除了 `operation_record` 上原有唯一索引 `uniq_biz_no`，但新增的只是普通组合索引 `idx_biz_no_biz_type`，没有恢复唯一约束。这样上线后相同业务键可被重复写入，容易引发幂等失效、重复操作记录和后续查询结果不确定。

    测试场景

    构造两条相同 `biz_no`、`biz_type` 的插入语句，期望数据库拒绝第二条；同时验证不同 `biz_type` 的相同 `biz_no` 是否符合业务预期。
