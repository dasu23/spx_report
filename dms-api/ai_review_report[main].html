major164-210数据一致性`routeDispatch` 现在允许 `ASSIGNED` 路线进入重派单流程，并在事务里无差别删除该路线的全部 dispatch 记录后再插入新的 `WAITING_ACCEPT` 记录。原先对 `ACCEPTED` dispatch 的保护被整段注释掉了，这会把司机已接受的派单直接覆盖掉，导致路线与司机确认状态回退、审计链丢失。恢复对已接受派单的保护，至少在删除前校验 `allDsipatches` 中不存在 `ACCEPTED` 记录；对重派单应采用逻辑失效旧记录而不是物理删除。示例：先 `Assert.isTrue(allDispatches.stream().noneMatch(dp -> dp.getStatus() == DispatchStatus.ACCEPTED), "Some routes have already been accepted...")`，然后仅将可重派的旧记录标记失效/逻辑删除，再新增新记录。构造一条 `RouteStatus.ASSIGNED` 且最新 `RouteDispatch.status=ACCEPTED` 的路线，调用 `/route/dispatch` 或 `/route/assign-driver` 后，应明确报错而不是把旧 dispatch 删除并改回 `WAITING_ACCEPT`。
