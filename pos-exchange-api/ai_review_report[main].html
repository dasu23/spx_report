major40-41配置变更`@Value("${pod.format:jpg}") private List<String> podFormats;` 在 Spring Boot 2.2 中会按逗号分隔绑定，但默认值只有 `jpg`，且后续比较未做大小写和空格归一化。配置成 `jpg, png` 或上传 `.JPG` 时，合法文件可能被错误拒绝，直接影响 POD 上传功能。在注入后统一标准化配置值，例如全部 `trim + lowerCase` 后再放入 `Set<String>`；同时将默认值改为显式列表并在配置文档中说明格式，例如 `@Value("#{'${pod.format:jpg,jpeg,png}'.split(',')}")` 配合初始化归一化。验证配置值分别为 `jpg,png`、`jpg, png`、`JPG,PNG` 时，`.jpg`、`.png`、`.JPG` 文件都能按预期通过。
major53-57边界条件这里直接对 `url` 执行 `substring(url.lastIndexOf(".") + 1)`，当 URL 为空、为 null、没有扩展名，或带查询串如 `a.jpg?x=1` 时，会得到错误的后缀甚至触发运行时异常，导致上传 POD 失败。先对 `url` 做非空和格式校验，再提取纯路径部分的扩展名，例如：`String path = StringUtils.substringBefore(url, "?"); int dot = path.lastIndexOf('.'); if (StringUtils.isBlank(path) || dot < 0 || dot == path.length() - 1) { throw new RRException("Invalid POD file url: " + url); } String ext = StringUtils.lowerCase(path.substring(dot + 1));`，再与白名单比较。补充用例覆盖 `null`、空字符串、无扩展名 URL、带 query string 的 URL、以及大写扩展名如 `.JPG` 的场景。
