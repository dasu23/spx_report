    修复建议

    将“存在性检查 + 创建容器”收敛到单个事务内，并对 chute 维度加锁；同时在数据库层补充能够约束同一 `facilityCode + sortId` 下唯一活动容器的唯一索引或幂等键，避免仅靠应用层先查后写。
