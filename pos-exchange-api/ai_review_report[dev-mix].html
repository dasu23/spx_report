    MAJORL57,74-75安全风险
    错误日志直接输出原始 `jsonString`（`message={}`），会把完整事件载荷写入日志。若消息中包含运单号、用户信息或后续扩展出的敏感字段，日志侧将形成敏感数据暴露面，且异常场景下更容易被集中采集与外传。
    修复建议与测试场景
    修复建议

    对日志做最小化与脱敏：仅记录 `eventId`、`type`、`status` 等必要字段，禁止打印完整消息体；解析失败场景可记录摘要/hash。示例：`log.error("[ParcelCounterConsumer] parse failed, payloadHash={}", DigestUtils.sha256Hex(jsonString), e);`。

    测试场景

    构造包含敏感字段（如 phone/token）的 Kafka 消息并触发异常，验证日志中不再出现原文敏感值，仅出现脱敏后的摘要或白名单字段。

    MAJORL59-72边界条件
