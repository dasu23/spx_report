    pos-service/src/main/java/com/miq/posservice/factory/pipeline/AbstractDataProducer.java1 Major1 个问题
    MAJORL43-47并发安全
    `finally` 中无条件执行 `queue.put(Collections.emptyList())`。当队列已满且消费者未启动、已退出或提交失败时，这里会永久阻塞，导致生产者线程无法结束，进而放大为整条流水线卡死。
    修复建议与测试场景
    修复建议

    避免在 `finally` 中使用无限阻塞的 `put`。可改为 `offer(poisonPill, timeout, unit)` 并在失败时记录错误后退出；或在放入 poison pill 前先处理中断/取消状态。例如：`if (!queue.offer(Collections.emptyList(), 5, TimeUnit.SECONDS)) { log.error("Failed to enqueue poison pill because queue is full"); }`。

    测试场景

    使用容量为 1 的队列，先放满数据且不启动 consumer，再让 `produce()` 正常结束或异常结束；验证 finally 不会永久阻塞在线程退出路径上。
