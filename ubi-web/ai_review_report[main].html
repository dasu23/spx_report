    MAJORL13安全风险
    原因： CreateApiKeyCmd.expiresInDays 新增后未添加任何范围校验，且注释明确允许为空表示长期有效，但没有限制非空时必须为正数或设置上限，导致创建 API Key 的请求体可以直接提交 0、负数或超大整数。影响： 一旦后续控制器/服务直接信任该 DTO，调用方可通过构造异常有效期绕过预期的密钥生命周期控制，造成永不过期、立即失效或整数边界异常等安全后果，尤其会削弱 API Key 轮换与过期治理。

    修复建议与测试场景
    修复建议

    @Min(value = 1, message = "expiresInDays must be greater than 0")
