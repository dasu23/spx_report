严重 pos-admin-api/src/main/java/com/miq/posadminapi/controller/sys/SSOLoginController.java:63
这里把 `req.getSession().getId()` 直接当作 `authToken` 拼进 302 Location。Session ID 属于服务端会话凭据，放入 URL 后会暴露在浏览器历史、代理日志、Referer 和前端埋点中，攻击者拿到该值后可直接复用会话，造成登录态劫持。
