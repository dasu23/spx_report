    CRITICALL139-140安全风险
    新增测试代码将 `innerApiKey` 以明文硬编码写入源码。即使该测试被 `@Disabled` 标记，凭证仍会进入版本库、CI 日志和代码分发链路，构成长期密钥泄露风险，符合 [SEC-002]。

    修复建议

    删除硬编码的真实 `innerApiKey`，改为从受控配置注入（如环境变量、测试密钥管理器或本地未入库配置文件）读取；若仅为集成测试占位，请使用明确的假值并确保不会指向真实凭证。

    测试场景

    补充检查：集成测试启动时仅允许从环境变量/外部配置读取 `innerApiKey`；当未提供配置时测试应跳过或失败，且源码库中不应出现真实 key/token 字面量。
