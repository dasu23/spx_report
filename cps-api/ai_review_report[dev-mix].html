    cps-infrastructure/src/main/java/com/speedx/cps/configuration/TransactionConfiguration.java1 Critical 1 Major2 个问题
    CRITICALL28-30数据一致性
    这里把 JDBC 的 `transactionManager(DataSource)` 标记为 `@Primary`，而新增的 `JpaTransactionManager` 既没有命名为 `transactionManager` 也不是主事务管理器。对 Spring Boot 2.5 + Spring Data JPA 而言，未显式指定事务管理器的 `@Transactional` 和部分仓储事务将优先落到 `DataSourceTransactionManager`，导致 JPA `EntityManager` 的 flush/rollback 不受管控。出现异常时可能发生部分数据已写入但未按预期回滚，属于事务边界错误带来的数据完整性风险。
    修复建议与测试场景
    修复建议
