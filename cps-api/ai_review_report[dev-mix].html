    cps-infrastructure/src/main/java/com/speedx/cps/config/BiApiConfig.java1 Major1 个问题
    MAJORL11-15安全风险
    `BiApiConfig` 使用了 Lombok `@Data`，而该类同时新增了敏感配置项 `innerApiKey`。`@Data` 会自动生成 `toString()`，一旦该配置对象被日志、异常信息或调试输出直接打印，`innerApiKey` 会以明文形式暴露，属于新增的敏感数据泄露风险。
    修复建议与测试场景
    修复建议

    不要对包含密钥的配置类使用 `@Data`。改为显式使用 `@Getter`/`@Setter`，并通过 `@ToString(onlyExplicitlyIncluded = true)` 白名单输出，或至少为 `innerApiKey` 增加 `@ToString.Exclude`，避免任何自动生成的字符串表示包含密钥。

    测试场景

    补充单元测试或日志安全测试：构造 `BiApiConfig` 并设置 `innerApiKey` 后，验证其 `toString()` 结果不包含密钥明文；同时检查异常/日志链路中打印该对象时不会泄露敏感字段。
