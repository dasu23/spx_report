严重 cps-infrastructure/src/main/java/com/speedx/cps/configuration/TransactionConfiguration.java:28-30
这里把 JDBC 的 `transactionManager(DataSource)` 标记为 `@Primary`，而新增的 `JpaTransactionManager` 既没有命名为 `transactionManager` 也不是主事务管理器。对 Spring Boot 2.5 + Spring Data JPA 而言，未显式指定事务管理器的 `@Transactional` 和部分仓储事务将优先落到 `DataSourceTransactionManager`，导致 JPA `EntityManager` 的 flush/rollback 不受管控。出现异常时可能发生部分数据已写入但未按预期回滚，属于事务边界错误带来的数据完整性风险。
💡 修复建议：将 `JpaTransactionManager` 设为默认事务管理器，并与注释保持一致，例如把其 bean 名改为 `transactionManager` 且标记 `@Primary`；若必须同时保留 JDBC 事务管理器，则为其使用独立名称（如 `jdbcTransactionManager`），并在仅需纯 JDBC/MyBatis 事务的方法上显式 `@Transactional(transactionManager = "jdbcTransactionManager")`。

新增的 `JpaTransactionManager` 没有被注册为默认事务管理器；相反，`transactionManager` 这个默认 bean 仍然是 `DataSourceTransactionManager`，这会让 `@Transactional` 和 Spring Data JPA 继续走 JDBC 事务，而不是 JPA 事务。涉及 JPA EntityManager 的提交、flush、回滚同步时，可能出现事务行为异常或持久化结果不一致。
💡 修复建议：将 JPA 事务管理器设为默认 bean，并把 `transactionManager` 名称绑定到 `JpaTransactionManager`。例如：`@Primary @Bean(name = "transactionManager") public PlatformTransactionManager transactionManager(EntityManagerFactory emf) { return new JpaTransactionManager(emf); }`；如果仍需保留 JDBC 事务管理器，请使用单独名称如 `dataSourceTransactionManager`，并仅在明确场景下通过 `@Transactional(transactionManager = "dataSourceTransactionManager")` 指定。
