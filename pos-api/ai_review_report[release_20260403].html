    MINORL88-90业务逻辑
    当 `customerRefId` 为空时直接用 `System.currentTimeMillis()` 生成值，无法保证并发下的唯一性。同一毫秒内的多个请求会得到相同 ID，而该字段注释明确要求唯一，可能导致下游 Beans 校验或创建请求偶发失败。
    修复建议与测试场景
    修复建议

    改为使用稳定的唯一 ID 生成方案，例如 UUID、雪花算法，或复用现有业务唯一流水号。示例：`req.setCustomerRefId(IdUtil.fastSimpleUUID());`。

    测试场景

    新增并发测试：同一毫秒内连续/并发构造多个 `customerRefId` 为空的请求，验证生成值均唯一。
