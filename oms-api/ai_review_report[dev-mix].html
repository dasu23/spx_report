    修复建议与测试场景
    修复建议

    将进度更新放到独立事务中提交，例如注入一个 `PROPAGATION_REQUIRES_NEW` 的 `TransactionTemplate`，或拆到单独的 Spring Bean 方法上并使用 `@Transactional(propagation = Propagation.REQUIRES_NEW)`：`requiresNewTx.execute(status -> { batchQueryTaskRepository.updateProgress(...); return null; });`。

    测试场景

    构造一个外层事务中先处理部分数据、再触发异常回滚的场景，断言异常后任务进度仍然被更新；同时验证当前实现下进度会被回滚。
