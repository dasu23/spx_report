严重 oms-domain/src/main/java/io/speedx/oms/domain/service/lastmile/LmCostCalculator.java:214-223
重量单位转换失败时仅记录日志并直接返回原始重量，后续会继续参与计费。只要 `fromUom/toUom` 配置错误或枚举不兼容，就会按错误单位计算 base cost，属于直接的资金计算错误，而且调用方无法感知本次计费已失真。
