    tracking-service/src/main/java/com/miq/trackingservice/api/SAPC4CApiClient.java1 Major1 个问题
    MAJORL252安全风险
    新增日志将 `patchBody` 直接输出到 error 级别：`log.error("SAP C4C patch case failed, caseId={}, body={}", caseUuid, patchBody, e);`。`patchBody` 是发往 SAP C4C 的更新请求体，通常可能包含工单备注、客户信息、联系方式或其他业务字段。失败场景下这些内容会被持久化到错误日志，扩大敏感业务数据暴露面，且 error 日志通常会被集中采集、长期保留并被更多运维/分析系统访问。
    修复建议与测试场景
    修复建议

    不要在错误日志中直接打印完整请求体。改为仅记录必要的定位字段（如 caseId、字段名白名单、请求体摘要/长度），并对可能包含个人信息的字段做脱敏。例如：`log.error("SAP C4C patch case failed, caseId={}, bodySize={}", caseUuid, patchBody == null ? 0 : patchBody.length(), e);`；如必须记录内容，先对备注、电话、邮箱等字段脱敏后再输出。

    测试场景

    构造一个包含姓名、电话、邮箱或备注内容的 `patchBody`，触发 patch 异常，验证 error 日志中不出现原始请求体明文，仅出现脱敏后的摘要信息。
