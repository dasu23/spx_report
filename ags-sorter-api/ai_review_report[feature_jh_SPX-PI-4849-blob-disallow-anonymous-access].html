    ags-sorter-domain/src/main/java/com/speedx/agssorter/domain/config/AzureBlobStorageConfig.java1 Major1 个问题
    MAJORL41-44API契约
    这里把 `azure.storage.blob-domain` 直接传给 `BlobServiceClientBuilder.endpoint(...)`，随后又调用 `getBlobContainerClient(containerName)`。但同一批改动里的 `BlobSasUtil` 注释和测试都把该配置解释为“包含容器名的完整容器 URL”。如果线上配置也按这个语义填写，SDK 会在已有容器 URL 后再次拼接 `containerName`，最终把 client 指向错误路径，导致上传/生成 SAS URL 失败。
    修复建议与测试场景
    修复建议

    统一配置语义并按 SDK 契约构建客户端：如果 `blob-domain` 是容器 URL，就应直接用 `new BlobContainerClientBuilder().endpoint(blobDomain).credential(...).buildClient()`；如果它只是账号级 endpoint，就保留 `BlobServiceClientBuilder`，但需把配置名/注释改成账号级地址并补充校验，避免把容器 URL 误传进来。

    测试场景

    使用 `azure.storage.blob-domain=https://<account>.blob.core.windows.net/<container>` 启动后，执行一次 `uploadExcelFile` 或 `upload99m`，验证实际访问 URL 不会出现 `/<container>/<container>/...`，且上传成功。

    日志直接打印 `uploadAndReturnSasUrl(...)` 返回的完整下载链接。SAS URL 含签名/令牌参数，写入 `info` 日志后会把临时访问凭证暴露给日志平台、运维人员或其他日志消费者。
    修复建议与测试场景
    修复建议

    禁止记录完整 SAS URL。日志中仅保留 blob 路径或做脱敏处理，例如移除查询串后再记录；如确需排障，单独记录对象 key、过期时间等非敏感信息。

    测试场景

    补充日志断言：上传成功后日志内容不应包含 `sig=`、`token=` 或完整查询串，只允许出现文件路径/文件名。

    ags-sorter-domain/src/main/java/com/speedx/agssorter/domain/service/BatchService.java1 Major1 个问题
    MAJORL145安全风险
    这里直接对 `request.getFileUrl()` 执行 `new URL(...).openStream()`，把前端可控 URL 作为服务端出站请求目标，形成 SSRF 面。攻击者可构造指向内网地址、云元数据地址或其他非 Azure 目标的 URL，借助服务端网络权限发起访问。
    修复建议与测试场景
    修复建议

    不要直接打开任意外部 URL。应在下载前严格校验 `fileUrl` 仅允许命中配置的 Azure Blob 域名/容器前缀，或改为只接收 blob 相对路径并通过受控 SDK/Blob 客户端读取对象。对协议、主机、端口和路径都要做白名单校验。

    测试场景

    补充用例：传入 `http://127.0.0.1:8080/...`、`http://169.254.169.254/...`、其他非白名单域名 URL，期望请求被拒绝且不会发生出站连接。

    ags-sorter-domain/src/main/java/com/speedx/agssorter/domain/utils/BlobSasUtil.java1 Major1 个问题
    MAJORL36-37边界条件
    `toRelativePath` 在 `stripped.equals(blobDomain)` 时也会进入该分支，但返回语句固定执行 `substring(blobDomain.length() + 1)`。当输入正好是容器根 URL 时，起始下标越过字符串末尾，直接抛出 `StringIndexOutOfBoundsException`。这是新增的边界条件缺陷。
    修复建议与测试场景
    修复建议

    把“刚好等于容器 URL”单独处理，例如：`if (stripped.equals(blobDomain)) { return ""; } if (stripped.startsWith(blobDomain + "/")) { return stripped.substring(blobDomain.length() + 1); }`，并补一条该场景的单测。

    测试场景

    调用 `BlobSasUtil.toRelativePath("https://spxblob.blob.core.windows.net/test-private-blob")`，期望不抛异常，并返回空字符串或约定的根路径表示。
